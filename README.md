# Lab 4 Qt20 — Multithreading in Qt

Учебный проект на **Qt / C++17**, посвящённый изучению многопоточности в Qt.

Проект опирается на методичку по лабораторной работе **«Практика 4_Qt20. Реализация многозадачности в Qt. Потоки»** и демонстрирует базовую архитектуру приложения, в котором несколько рабочих объектов формируют точки в параллельных потоках, а GUI-поток отображает результат.

## Цель проекта

Изучить и сравнить несколько способов организации многопоточности в Qt:

- `QThread`
- `QtConcurrent::run()`
- `QRunnable` / `QThreadPool`
- синхронизацию через `QMutex`, `QSemaphore`, `QSystemSemaphore`

## Что есть в репозитории сейчас

На текущем этапе в репозитории подготовена основа проекта для части с **Worker + QThread**:

- выделена библиотека **`workers_core`** с базовыми классами модели и логики;
- выделена библиотека **`workers_widgets`** с виджетом визуализации;
- создано GUI-приложение на базе `QMainWindow`;
- реализован рабочий объект `Worker`, который:
  - использует общую координату `X`;
  - имеет собственные `Y` и цвет;
  - формирует заданное количество точек;
  - отправляет точки в GUI через сигнал `signalAddPoint(MyPoint)`;
  - завершает работу сигналом `endWork()`;
- реализован класс `PointsWidget`, который хранит точки и отрисовывает их на экране;
- добавлена структура `WorkersRunSettings` для параметров одного запуска worker-объектов.

## Архитектура

### `workers_core`

Библиотека содержит общие классы предметной области:

- **`MyPoint`** — лёгкий value-object для хранения координаты и цвета точки;
- **`Worker`** — `QObject`-наследник, выполняющий полезную работу и генерирующий точки;
- **`WorkersRunSettings`** — набор параметров одного запуска worker-объектов.

### `workers_widgets`

Библиотека содержит UI-компонент:

- **`PointsWidget`** — виджет, который накапливает `MyPoint` и отображает их как цветные вертикальные линии.

### `app`

GUI-приложение с главным окном:

- создаёт и запускает worker-объекты в отдельных потоках `QThread`;
- получает точки из рабочих потоков через сигналы/слоты;
- отображает результат в `PointsWidget`;
- очищает сцену и подготавливает повторный запуск.

### `tests`

Папка для модульных тестов проекта.

## Сборка

### Требования

- Qt 5.x или Qt 6.x
- CMake 3.16+
- Компилятор с поддержкой C++17

### Сборка через CMake

```bash
cmake -S . -B build
cmake --build build --parallel
```

### Запуск тестов

```bash
ctest --test-dir build --output-on-failure
```

## Что демонстрирует проект

Во время работы несколько worker-объектов формируют координаты точек в отдельных потоках. GUI-поток принимает точки через механизм сигналов/слотов и передаёт их в виджет отображения. Для большей наглядности в worker-объектах используется программная задержка.

На Windows в `main.cpp` можно дополнительно зафиксировать выполнение процесса на одном логическом ядре через `SetProcessAffinityMask(...)`, чтобы нагляднее наблюдать конкуренцию потоков в рамках лабораторной работы.
